# 广告系统设计文档

> 适用版本：NativeBridge 代理模式（Google Mobile Ads SDK 已停用）

---

## 一、架构概览

```
业务层 (GameManager / UI)
        ↓ 调用
  AdsManager              ← 唯一广告入口，封装所有广告逻辑
        ↓ 调用 / 订阅
NativeBridgeManager       ← 平台通信层，负责 Android / iOS 原生交互
        ↕ JSON 消息
   原生平台 (Java / ObjC)
```

**核心原则**：所有广告展示必须经过 `AdsManager`，禁止外部直接调用 `NativeBridgeManager` 的广告相关方法。

---

## 二、关键文件

| 文件 | 职责 |
|------|------|
| `Controller/AdsManager.cs` | 广告总入口，管理请求状态、回调路由、防重入 |
| `Controller/NativeBridge/NativeBridgeManager.cs` | 原生通信层，发送 JSON 消息，触发结果事件 |
| `Controller/UI/Layers/AdsLayerUI.cs` | 广告确认弹窗，发起广告请求并处理 UI 结果 |
| `Controller/UI/GameLayerMediator.cs` | 弹窗调度中介，封装 UILayerManager 的 Show/Hide |

---

## 三、广告类型

| 类型 | 入口方法 | adType 值 | 说明 |
|------|---------|-----------|------|
| 激励视频 - 免广告 | `AdsManager.NoAdsAction()` | 0 | 观看后本局免广告 |
| 激励视频 - 获得撤销 | `AdsManager.ShowGetUndoAction()` | 0 | 观看后获得撤销次数 |
| 插屏广告 | `AdsManager.ShowInterstitial()` | 1 | 无奖励，由计数器或定时触发 |
| Banner | `AdsManager.ShowBanner()` | — | 当前 NativeBridge 模式下暂未实现 |

## 四、按钮点击完整流程

### 4.1 流程图

```
用户点击 WatchButton
        │
        ▼
AdsLayerUI.OnWatchAdsClicked()
        │
        ├─ GetUndo → _adsManager.ShowGetUndoAction(onSuccess, onFailed)
        └─ NoAds   → _adsManager.NoAdsAction(onSuccess, onFailed)
                                        │
                    ┌───────────────────┘
                    ▼
          _isAdPlaying == true?
                    │
          ┌─── Yes ─┴─ No ────────────────────────┐
          ▼                                         ▼
    onFailed()                            _lastShowingType  = type
 ShowRewardResult                         _pendingOnSuccess = onSuccess
(DID_NOT_LOADED)                          _pendingOnFailed  = onFailed
     [END]                                _isAdPlaying      = true
                                          StartCoroutine(LoadRewardedVideo)
                                                    │
                                  ┌─────────────────┘
                                  │  【协程逐帧执行，不阻塞主线程】
                                  ▼
                        _isHandeldAction? → AdsHandheld.Show()
                                  │
                                  ▼
                        while (!IsInitSuccess() && waited < 3f)
                            waited += deltaTime
                            yield return null
                                  │
                                  ▼ （初始化完成 或 等待超时）
                        _isHandeldAction? → AdsHandheld.Hide()
                                  │
                                  ▼
                        IsADReady(RewardVideo)?
                                  │
                        ┌── No ───┴── Yes ──────────────────┐
                        ▼                                     ▼
              InvokeAndClearCallbacks                SendMessageToPlatform
                   (success: false)                  (ShowVideo, adType=0)
                        │                                     │
                        │                              【协程结束】
                        │                          主线程正常运行
                        │                                     │
                        │                   ══════════════════╪══════════════════
                        │                         原生层播放广告
                        │                   ══════════════════╪══════════════════
                        │                                     │
                        │                       ┌─────────────┼──────────────┐
                        │                    成功回调       失败回调
                        │                       │             │
                        │                ADPlayResult    ADPlayResult
                        │               (success=true)  (success=false)
                        │                       │             │
                        │                OnVideoPlayEnd  OnVideoPlayFailed
                        │                  (adType=0)      (adType=0)
                        │                       │             │
                        │                Handle...End   Handle...Failed
                        │                adType==0?✓    adType==0?✓
                        │                       │             │
                        │                OnRewardedUser()     │
                        │                (发放奖励)           │
                        │                       │             │
                        └───────────────────────┴─────────────┘
                                                │
                                                ▼
                                  InvokeAndClearCallbacks(success)
                                                │
                                  _pendingOnSuccess / _pendingOnFailed → 局部变量
                                  _lastShowingType = None
                                  _isAdPlaying     = false
                                                │
                                  ┌── true ─────┴───── false ──┐
                                  ▼                              ▼
                            onSuccess()                    onFailed()
                          UILayerManager               ShowRewardResult
                          .Hide(AdsLayer)             (DID_NOT_LOADED)
                               [END]                        [END]
```

### 4.2 两条结束路径

| 路径 | 触发条件 | 结果 |
|------|---------|------|
| **成功** | 原生 `ADPlayResult` isSuccess=true | 发放奖励 → 关闭弹窗 |
| **失败** | 原生 `ADPlayResult` isSuccess=false | 显示失败提示 |

两条路径均通过 `InvokeAndClearCallbacks` 统一出口，状态在此处原子性清零。

## 五、状态机

`AdsManager` 内部维护以下状态字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `_isAdPlaying` | `bool` | 防重入标志，广告进行中为 `true` |
| `_lastShowingType` | `RewardAdsType` | 当前请求的广告奖励类型 |
| `_pendingOnSuccess` | `Action` | 本次调用的成功回调 |
| `_pendingOnFailed` | `Action` | 本次调用的失败回调 |

**状态转换：**

```
[空闲]
  │ ShowGetUndoAction() / NoAdsAction()
  ▼
[播放中] _isAdPlaying=true
  │ 成功 / 失败
  ▼
[空闲] _isAdPlaying=false  （由 InvokeAndClearCallbacks 统一重置）
```

## 六、事件流（跨对象）

```
NativeBridgeManager
  ├─ static event Action<int> OnVideoPlayEnd     → AdsManager.HandleNativeBridgeVideoEnd(int)
  └─ static event Action<int> OnVideoPlayFailed  → AdsManager.HandleNativeBridgeVideoFailed(int)

AdsManager
  ├─ 订阅：OnEnable()   → += 两个事件
  └─ 取消：OnDisable()  → -= 两个事件
```

`HandleNativeBridgeVideoEnd` 和 `HandleNativeBridgeVideoFailed` 均校验 `adType == RewardVideo(0)`，插屏广告（adType=1）的回调被安全忽略，不会误触发奖励。

---

## 七、插屏广告流程

插屏广告与激励视频共用协程模式，但无奖励回调：

```
ShowInterstitial()
  │ 防卫：IsHasKeyNoAds() || !_isIntersitial → return
  │
  └─ StartCoroutine(LoadInterstitialVideo)
              │
              ├─ AdsHandheld.Show()
              ├─ 等待 IsInitSuccess()（最多 3 秒）
              ├─ AdsHandheld.Hide()
              │
              ├─ IsADReady(Interstitial)?
              │     ├─ No  → LogWarning，结束
              │     └─ Yes → SendMessageToPlatform(ShowVideo, adType=1)
              │
              └─ 协程结束（无超时，无奖励回调）
```

插屏结果通过 `OnVideoPlayEnd(adType=1)` 回调，`HandleNativeBridgeVideoEnd` 因 `adType != 0` 直接返回，不影响激励视频状态。

触发方式：

| 方式 | 说明 |
|------|------|
| `ShowInterstitial()` | 直接触发 |
| `TryShowIntersitialByCounter()` | 计数达到 `_intersitialCallsBorder`（默认3）后触发 |
| `InvokeRepeating(ShowInterstitial)` | Inspector 开启 `_intersitialRepeatCall` 后定时触发 |

## 八、调用规范

### 正确用法

```csharp
// 激励视频：由 AdsLayerUI 发起，传入结果回调
_adsManager.ShowGetUndoAction(
    onSuccess: () => UILayerManager.Instance.Hide(GameLayerMediator.AdsLayer),
    onFailed:  () => ShowRewardResult(RewardAdsState.DID_NOT_LOADED)
);

// 插屏广告：由游戏逻辑触发，无回调
_adsManager.TryShowIntersitialByCounter();

// 免广告检测
if (_adsManager.IsHasKeyNoAds()) { /* 隐藏广告入口 */ }
```

### 禁止事项

- **禁止**直接调用 `NativeBridgeManager.Instance.SendMessageToPlatform(ShowVideo, ...)`
- **禁止**在 `AdsManager` 以外订阅 `NativeBridgeManager.OnVideoPlayEnd`
- **禁止**在广告播放中再次调用 `NoAdsAction()` / `ShowGetUndoAction()`（会被防重入拦截并触发 `onFailed`）

## 九、Inspector 配置

| 字段 | 说明 |
|------|------|
| `_isIntersitial` | 是否启用插屏广告 |
| `_isBanner` | 是否启用 Banner（当前无效） |
| `_isReward` | 是否启用激励视频 |
| `_intersitialRepeatCall` | 是否启动时开启插屏定时轮播 |
| `_firstCallIntersitialTime` | 轮播首次触发延迟（秒） |
| `_repeatIntersitialTime` | 轮播间隔（秒） |
| `_isHandeldAction` | 等待广告时是否显示系统加载指示器 |
| `_intersitialCallsBorder` | `TryShowIntersitialByCounter` 触发阈值 |

## 十、已知限制

| 限制 | 说明 |
|------|------|
| Banner 未实现 | `ShowBanner()` 为空实现，`BannerHeight` 始终为 0 |
| 无超时保护 | 激励视频和插屏广告均未设置超时保护，若原生层无回调会导致状态永久锁定 |
| `AdsManager` 被禁用时回调丢失 | `OnDisable` 会取消事件订阅，若广告播放期间对象被禁用则回调永远不会触发 |
