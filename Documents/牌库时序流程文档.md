## 文档信息

- **创建日期**: 2026-03-23
- **文档版本**: v1.0
- **适用模块**: WordSolitaire 牌库 (PackDeck)

## 1. 牌库概述

### 1.1 牌库定义

牌库（PackDeck）是WordSolitaire游戏的初始卡牌来源，存储所有未发出的卡牌。牌库中的卡牌以卡背朝上的方式堆叠显示。

### 1.2 节点结构

```
PackDeck (WordSolitaireDeck)
├── BackGround (Image, 卡背背景)
│   └── Localized Text (LocalizedText)  ← "恢复库存"文本
├── CardBackStack (RectTransform)       ← 卡背堆叠容器
└── CountLabelBack (Image)              ← 数量标签背景
    └── CountLabel (Text)               ← 数量显示
```

### 1.3 核心属性

- **DeckType**: `WordDeckType.Pack`（牌库类型）
- **CardsArray**: 存储牌库中的所有卡牌
- **HasCards**: 判断牌库是否有牌
- **CardsCount**: 牌库中卡牌的数量

## 2. 初始化流程

### 2.1 关卡初始化时序

```
关卡初始化开始
    ↓
WordSolitaireCardLogic.InitializeLevel()
    ↓
生成所有卡牌（分类卡 + 单词卡 + 万能卡）
    ↓
按照关卡布局分配卡牌到各列区
    ↓
剩余卡牌放入牌库（PackDeck.PushCard）
    ↓
PackDeck.UpdateCardsPosition(false)
    ↓
UpdatePackCardsPosition()
    ↓
更新牌库卡牌位置（错位堆叠）
    ↓
UpdateRestoreTextVisibility()
    ↓
显示 CardBackStack 和 CountLabelBack
    ↓
初始化完成
```

### 2.2 初始化详细说明

1. **卡牌生成**
   - 根据关卡配置生成所有需要的卡牌
   - 卡牌初始状态为卡背朝上（`shouldFaceUp: false`）

2. **卡牌分配**
   - 部分卡牌直接分配到列区（Column）
   - 剩余卡牌全部放入牌库

3. **牌库堆叠**
   - 使用 `DECK_SPACE_VERTICAL_BOTTOM_CLOSED` 间距（默认2f）
   - 每张牌Y轴向下偏移 `i * packSpace` 像素
   - 所有卡牌Z轴相同，无偏移
   - 通过 `SetSiblingIndex(i)` 控制渲染层级

4. **UI显示**
   - CardBackStack：显示（牌库有卡牌时）
   - CountLabelBack：显示，CountLabel显示卡牌数量
   - "恢复库存"文本：隐藏

## 3. 发牌流程

### 3.1 发牌时序

```
用户点击牌库
    ↓
WordSolitaireDeck.OnPointerClick()
    ↓
检查是否点击"恢复库存"文本
    ↓（否）
调用 base.OnPointerClick(eventData)
    ↓
WordSolitaireCardLogic.HandlePackClick()
    ↓
判断牌库状态
    ↓（牌库有牌）
PackDeck.Pop()  // 从牌库移除一张牌
    ↓
HandDeck.PushCard(card, true, 1)  // 放入手牌区，正面朝上
    ↓
HandDeck.UpdateCardsPosition(false)
    ↓
PackDeck.UpdateCardsPosition(false)
    ↓
UpdatePackCardsPosition()
    ↓
更新牌库堆叠位置
    ↓
UpdateRestoreTextVisibility()
    ↓
检查牌库是否耗尽
    ↓（否）
更新 CountLabel 显示数量
    ↓
播放发牌音效
    ↓
消耗步数（-1）
    ↓
发牌完成
```

### 3.2 发牌详细说明

1. **点击检测**
   - 用户点击牌库区域
   - 判断是否点击"恢复库存"文本（通过 `RectTransformUtility.RectangleContainsScreenPoint`）
   - 如果点击"恢复库存"文本，执行恢复库存流程

2. **发牌操作**
   - `PackDeck.Pop()`: 从牌库中移除最后一张牌（索引最大的牌）
   - `HandDeck.PushCard(card, true, 1)`: 将卡牌放入牌库，设置为正面朝上

3. **位置更新**
   - 牌库：重新计算所有卡牌的堆叠位置
   - 手牌区：根据 `HandDirection` 和 `MaxVisibleCards` 更新手牌位置

4. **UI更新**
   - CardBackStack：保持显示（牌库仍有卡牌）
   - CountLabelBack：保持显示，CountLabel更新为新的数量
   - "恢复库存"文本：保持隐藏

5. **事件触发**
   - 播放 `MoveToWaste` 音效
   - 消耗1步
   - 如果牌库耗尽，发布 `PackEmpty` 事件

## 4. 牌库耗尽流程

### 4.1 牌库耗尽时序

```
最后一张牌被发出手牌区
    ↓
PackDeck.CardsCount == 0
    ↓
PackDeck.UpdateCardsPosition(false)
    ↓
UpdatePackCardsPosition()
    ↓
UpdateRestoreTextVisibility()
    ↓
CardBackStack.SetActive(false)
    ↓
CountLabelBack.SetActive(false)
    ↓
restoreText.SetActive(true)  // 显示"恢复库存"文本
    ↓
发布 GameEventBus.PublishPackEmpty() 事件
    ↓
牌库耗尽完成
```

### 4.2 牌库耗尽详细说明

1. **UI状态切换**
   - CardBackStack：隐藏（牌库无卡牌）
   - CountLabelBack：隐藏
   - "恢复库存"文本：显示

2. **事件发布**
   - 发布 `PackEmpty` 事件
   - 订阅该事件的组件可以响应牌库耗尽状态

## 5. 恢复库存流程

### 5.1 恢复库存时序

```
用户点击"恢复库存"文本
    ↓
WordSolitaireDeck.OnPointerClick()
    ↓
检查 DeckType == WordDeckType.Pack
    ↓
检查 restoreText.gameObject.activeSelf == true
    ↓
检查点击位置在 restoreText 范围内
    ↓
调用 WordSolitaireCardLogic.RestorePackCards()
    ↓
获取手牌区所有卡牌（从旧到新）
    ↓
从手牌区移除所有卡牌（HandDeck.RemoveCard）
    ↓
按初始顺序放回牌库
    ↓
PackDeck.PushCard(card, false, 0)  // 放入牌库，卡背朝上
    ↓
HandDeck.UpdateCardsPosition(false)  // 手牌区清空
    ↓
PackDeck.UpdateCardsPosition(false)
    ↓
UpdatePackCardsPosition()
    ↓
更新牌库堆叠位置
    ↓
UpdateRestoreTextVisibility()
    ↓
CardBackStack.SetActive(true)
    ↓
CountLabelBack.SetActive(true)
    ↓
CountLabel.text = CardsCount.ToString()
    ↓
restoreText.SetActive(false)
    ↓
播放恢复音效
    ↓
消耗步数（-1）
    ↓
恢复库存完成
```

### 5.2 恢复库存详细说明

1. **点击检测**
   - 只在牌库耗尽时，"恢复库存"文本显示时生效
   - 通过 `RectTransformUtility.RectangleContainsScreenPoint` 精确判断点击位置

2. **卡牌迁移**
   - 从手牌区获取所有卡牌（保持原始顺序，从旧到新）
   - 从手牌区移除所有卡牌
   - 按相同顺序放回牌库（最旧的牌在最底部）

3. **位置更新**
   - 牌库：重新计算所有卡牌的堆叠位置
   - 手牌区：清空后更新位置（无卡牌）

4. **UI状态切换**
   - CardBackStack：显示（牌库恢复有卡牌）
   - CountLabelBack：显示，CountLabel显示恢复后的数量
   - "恢复库存"文本：隐藏

5. **事件触发**
   - 播放 `MoveToPack` 音效
   - 消耗1步（根据产品文档第108行和第229行要求）

## 6. 牌库卡牌堆叠逻辑

### 6.1 堆叠计算

```csharp
private void UpdatePackCardsPosition()
{
    // 获取牌库堆叠间距（使用垂直封闭间距）
    float packSpace = CardLogicComponent?.GetSpaceFromDictionary(DeckSpacesTypes.DECK_SPACE_VERTICAL_BOTTOM_CLOSED) ?? 2f;
    Vector3 basePosition = transform.position;

    for (int i = 0; i < CardsCount; i++)
    {
        // 错位堆叠：每张牌向下移动一点，实现视觉堆叠效果
        Vector3 newPosition = new Vector3(
            basePosition.x,
            basePosition.y - i * packSpace,  // Y轴：向下错位
            basePosition.z
        );
        CardsArray[i].SetPosition(newPosition);
        CardsArray[i].transform.SetSiblingIndex(i);
    }

    // 更新"恢复库存"文本显示
    UpdateRestoreTextVisibility();
}
```

### 6.2 堆叠规则

| 索引 (i) | 位置计算 | 视觉效果 |
|----------|----------|----------|
| 0 | basePosition.y - 0 * packSpace | 最上层（最底部显示） |
| 1 | basePosition.y - 1 * packSpace | 第2层 |
| 2 | basePosition.y - 2 * packSpace | 第3层 |
| ... | ... | ... |
| n-1 | basePosition.y - (n-1) * packSpace | 最下层（最顶部显示） |

### 6.3 视觉效果说明

1. **错位堆叠**
   - 每张牌向下偏移2像素（默认）
   - 形成类似真实牌堆的视觉厚度效果
   - 通过 `SetSiblingIndex` 控制渲染顺序

2. **层级关系**
   - 索引小的牌在上方（视觉上）
   - 索引大的牌在下方（视觉上）
   - 发牌时总是取索引最大的牌（最上面的牌）

3. **发牌顺序**
   - 第一次发牌：取索引 `CardsCount-1` 的牌
   - 第二次发牌：取索引 `CardsCount-2` 的牌
   - 以此类推，直到牌库耗尽

## 7. UI显示逻辑

### 7.1 UI元素状态

| 牌库状态 | CardBackStack | CountLabelBack | CountLabel | "恢复库存"文本 |
|----------|---------------|----------------|-------------|----------------|
| 有牌 | 显示 | 显示 | 显示数量（如"52"） | 隐藏 |
| 无牌 | 隐藏 | 隐藏 | 不显示 | 显示 |

### 7.2 CountLabel更新

```csharp
private void UpdateRestoreTextVisibility()
{
    // 只对牌库类型执行此逻辑
    if (DeckType != WordDeckType.Pack) return;

    // 背景状态：有牌时显示元素，无牌时隐藏
    bool shouldShow = HasCards;

    // 查找并控制各子元素的显示状态
    Transform cardBackStack = transform.Find("CardBackStack");
    if (cardBackStack != null)
    {
        cardBackStack.gameObject.SetActive(shouldShow);
    }

    Transform countLabelBack = transform.Find("CountLabelBack");
    if (countLabelBack != null)
    {
        countLabelBack.gameObject.SetActive(shouldShow);

        // 更新CountLabel文本，显示牌库中卡牌的数量
        Transform countLabel = countLabelBack.Find("CountLabel");
        if (countLabel != null)
        {
            Text countLabelText = countLabel.GetComponent<Text>();
            if (countLabelText != null)
            {
                countLabelText.text = CardsCount.ToString();
            }
        }
    }

    Transform restoreText = transform.Find("BackGround/Localized Text");
    if (restoreText != null)
    {
        // "恢复库存"文本：有牌时隐藏,无牌时显示（与背景元素相反）
        restoreText.gameObject.SetActive(!shouldShow);
    }
}
```

### 7.3 UI更新时机

1. **关卡初始化**
   - 显示 CardBackStack 和 CountLabelBack
   - CountLabel 显示初始数量

2. **发牌操作**
   - 每次发牌后更新 CountLabel
   - 保持 CardBackStack 和 CountLabelBack 显示

3. **牌库耗尽**
   - 隐藏 CardBackStack 和 CountLabelBack
   - 显示"恢复库存"文本

4. **恢复库存**
   - 显示 CardBackStack 和 CountLabelBack
   - CountLabel 显示恢复后的数量
   - 隐藏"恢复库存"文本

## 8. 事件流程汇总

### 8.1 完整游戏流程

```
游戏开始
    ↓
牌库初始化（显示堆叠和数量）
    ↓
用户点击牌库发牌
    ↓
发牌 → 更新数量 → 消耗步数
    ↓
重复发牌，直到牌库耗尽
    ↓
显示"恢复库存"文本
    ↓
用户点击"恢复库存"文本
    ↓
恢复库存 → 显示堆叠和数量 → 消耗步数
    ↓
重复发牌和恢复库存循环
    ↓
游戏结束（胜利或失败）
```

### 8.2 关键决策点

| 决策点 | 条件 | 操作 |
|--------|------|------|
| 点击牌库 | 牌库有牌 | 发牌（消耗1步） |
| 点击牌库 | 牌库无牌，手牌有牌 | 恢复库存（消耗1步） |
| 点击牌库 | 牌库无牌，手牌无牌 | 播放错误音效 |
| 点击"恢复库存"文本 | 文本可见 | 恢复库存（消耗1步） |

- ✅ 点击牌库发牌必须消耗1步
- ✅ 每次从牌库顺序抽出1张手牌至手牌区
- ✅ 牌库数量-1，步数-1

- ✅ 操作效果：将手牌区所有手牌按初始顺序放回牌库并恢复为卡背状态

## 10. 注意事项

### 10.1 开发注意事项

1. **节点结构**
   - 必须参考 `WordSolitaire 场景节点说明.md` 文档
   - CardBackStack 和 CountLabelBack 是 BackGround 的兄弟节点，不是子节点

2. **代码规范**
   - 所有代码修改后必须刷新Unity并验证编译
   - 确保Console中无error
   - 添加中文注释说明代码逻辑

3. **产品规则**
   - 发牌和恢复库存都消耗1步
   - 恢复库存必须保持初始顺序
   - UI显示状态必须与牌库状态同步

### 10.2 测试检查点

1. **初始化测试**
   - 牌库显示正确的卡牌数量
   - CardBackStack 和 CountLabelBack 正确显示
   - 卡牌堆叠位置正确（Y轴错位）

2. **发牌测试**
   - 点击牌库正确发牌
   - CountLabel 实时更新数量
   - 发牌后牌库堆叠正确更新

3. **牌库耗尽测试**
   - 最后一张牌发完后UI状态正确切换
   - "恢复库存"文本正确显示
   - CardBackStack 和 CountLabelBack 正确隐藏

4. **恢复库存测试**
   - 点击"恢复库存"文本正确触发
   - 手牌区卡牌正确放回牌库
   - 卡牌顺序保持初始顺序
   - UI状态正确切换回有牌状态

5. **边界测试**
   - 牌库和手牌都空时点击牌库，播放错误音效
   - 快速连续点击牌库，不出现错误
