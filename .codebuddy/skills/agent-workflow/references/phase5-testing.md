# 阶段5：测试验证 - 详细指南

## 负责代理
`unity-developer` + `unity-performance`

## 目标
全面测试功能实现，确保功能正常、性能良好、与现有系统集成无误。

## 测试类型

### 1. 功能测试（必须）

#### 1.1 基本功能测试
测试核心功能是否按预期工作。

**测试方法**：
- 在Unity编辑器中运行
- 执行功能的基本操作
- 验证输出是否正确

**测试用例示例**：
```
功能：多语言支持系统
用例1：切换语言
- 操作：调用SwitchLanguage("en")
- 预期：UI文本切换为英文
- 结果：[通过/失败]

用例2：加载语言文件
- 操作：启动游戏
- 预期：自动加载默认语言
- 结果：[通过/失败]
```

#### 1.2 边界测试
测试极端情况和边界值。

**常见边界情况**：
- 空值：参数为null时的处理
- 空集合：列表为空时的处理
- 最大值：数值达到极限时
- 最小值：数值为0或负数时
- 特殊输入：非法字符、超长字符串

**测试示例**：
```csharp
// 边界测试
public void TestBoundaries()
{
    // 测试null
    service.DoSomething(null);  // 应该不崩溃

    // 测试空集合
    service.ProcessList(new List<int>());  // 应该正常处理

    // 测试极限值
    service.SetValue(int.MaxValue);  // 应该正确处理

    // 测试负数
    service.SetValue(-1);  // 应该验证或拒绝
}
```

#### 1.3 异常测试
测试错误处理是否正确。

**测试场景**：
- 文件不存在
- 网络断开
- 资源加载失败
- 无效的用户输入

**检查清单**：
- [ ] 是否有适当的错误提示？
- [ ] 是否记录错误日志？
- [ ] 是否有恢复机制？
- [ ] 是否不会崩溃？

### 2. 集成测试（必须）

#### 2.1 与现有系统集成
验证新功能与现有系统的兼容性。

**测试要点**：
- [ ] 是否影响现有功能？
- [ ] 事件系统是否正常工作？
- [ ] 数据流是否正确？
- [ ] 依赖注入是否正确？

#### 2.2 场景测试
在实际游戏场景中测试。

**测试步骤**：
1. 在Unity中打开主场景
2. 运行游戏
3. 执行包含新功能的完整流程
4. 观察是否有异常

#### 2.3 多场景测试（如适用）
如果涉及多个场景，测试场景切换。

**检查清单**：
- [ ] 场景加载时是否正确初始化？
- [ ] 场景卸载时是否正确清理？
- [ ] 数据是否正确持久化？
- [ ] 事件订阅是否正确管理？

### 3. 性能测试（重要）

#### 3.1 帧率测试
使用`unity-performance`代理进行性能分析。

**测试方法**：
```csharp
// 使用性能监控器
public class PerformanceTest
{
    void Start()
    {
        // 开启Profiler
        Profiler.enabled = true;

        // 执行功能
        TestFeature();

        // 检查统计
        Debug.Log($"FPS: {1.0f / Time.deltaTime}");
        Debug.Log($"Memory: {Profiler.GetTotalAllocatedMemoryLong() / 1024 / 1024}MB");
    }
}
```

**性能指标**：
- [ ] 帧率是否稳定？（目标：60FPS）
- [ ] 是否有明显的帧率下降？
- [ ] CPU使用是否合理？
- [ ] GPU使用是否合理？

#### 3.2 内存测试
检查内存使用和泄漏。

**测试方法**：
1. 运行功能前记录内存
2. 重复执行功能多次
3. 记录内存增长
4. 检查是否有内存泄漏

**检查清单**：
- [ ] 内存使用是否合理？
- [ ] 是否有内存泄漏？
- [ ] GC频率是否正常？
- [ ] GC分配是否过多？

**使用Unity Memory Profiler**：
```
Window > Analysis > Memory Profiler
- Take Snapshot（执行功能前）
- 执行功能
- Take Snapshot（执行功能后）
- 对比两次快照
```

#### 3.3 性能分析
使用Unity Profiler进行详细分析。

**分析步骤**：
```
1. 打开Profiler（Window > Analysis > Profiler）
2. 运行游戏并执行功能
3. 记录性能数据
4. 分析CPU使用（查看耗时函数）
5. 分析Rendering（查看Draw Call）
6. 分析Memory（查看GC Alloc）
```

**关注指标**：
- CPU Usage：主线程时间
- Rendering：Draw Call数量
- Memory：GC Alloc大小
- Scripts：脚本执行时间

### 4. 平台测试（如适用）

#### 4.1 目标平台测试
在实际目标平台上测试。

**常见平台**：
- Windows/Mac/Linux
- Android
- iOS
- WebGL

**平台特定检查**：
- [ ] 是否在目标平台正常运行？
- [ ] 性能是否满足要求？
- [ ] 是否有平台特定问题？
- [ ] 输入系统是否正常？（触摸/鼠标/键盘）

#### 4.2 不同设备测试
在不同配置的设备上测试。

**测试设备**：
- 高端设备
- 中端设备
- 低端设备

### 5. 回归测试（重要）

#### 5.1 现有功能测试
确保新功能没有破坏现有功能。

**测试方法**：
- 运行关键功能的测试用例
- 验证现有功能是否正常
- 检查是否有意外的副作用

**检查清单**：
- [ ] 游戏启动是否正常？
- [ ] 核心玩法是否受影响？
- [ ] UI是否正常显示？
- [ ] 数据保存/加载是否正常？

## 测试通过标准

### ✅ 完全通过
- 所有功能测试通过
- 无高优先级问题
- 性能满足要求
- 回归测试通过

### ⚠️ 有条件通过
- 核心功能正常
- 有低优先级问题
- 性能基本满足要求

### ❌ 不通过
- 核心功能失败
- 有高优先级问题
- 性能不达标
- 破坏现有功能
